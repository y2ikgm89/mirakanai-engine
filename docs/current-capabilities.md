# Current Capabilities

This page is the human-readable summary of the current GameEngine capability contract. The machine-readable source of truth is `engine/agent/manifest.json`; use `pwsh -NoProfile -ExecutionPolicy Bypass -File tools/agent-context.ps1` when an agent needs exact fields, recipe ids, command surfaces, host gates, or validation commands.

## Ready Foundations

- Validation tooling includes strict clang-tidy compile database enforcement for the default Windows Visual Studio `dev` preset and the Linux `ci-linux-tidy` preset used by CI static analysis: `pwsh -NoProfile -ExecutionPolicy Bypass -File tools/check-tidy.ps1` verifies `.clang-tidy`, requests/configures CMake File API codemodel data when native `compile_commands.json` is unavailable, synthesizes a clang-tidy-compatible compile database when needed, supports targeted changed-file analysis through `-Files`, and reports missing CMake or clang-tidy as local tool blockers.
