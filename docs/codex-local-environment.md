# Codex Local Environment

Codex app local environments are useful for this repository, but the setup should stay lightweight. Use the local environment to prepare new Codex worktrees and expose common repository commands as Codex app actions. Do not run dependency bootstrap or full validation automatically on every new worktree.

Official Codex app documentation: <https://developers.openai.com/codex/app/local-environments>.

## Setup Script

Use this Windows setup script in the Codex app local environment:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File tools/codex-local-environment-setup.ps1
```

The script:

- links `external/vcpkg` into the new worktree when the source checkout already has it;
- links `vcpkg_installed` into the new worktree when the source checkout already has it;
- runs `tools/check-toolchain.ps1` in the worktree.

It intentionally does not clone vcpkg, run `tools/bootstrap-deps.ps1`, run `tools/validate.ps1`, install package-manager dependencies, or delete cleanup paths. Those steps are either heavy, network-dependent, approval-gated, or task-specific.

If the Codex app runs the setup script from a directory other than the source checkout, pass the roots explicitly:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File tools/codex-local-environment-setup.ps1 -SourceRoot G:\workspace\development\GameEngine -WorktreeRoot $env:CODEX_WORKTREE_PATH
```

## Actions

Actions are manual shortcuts in the Codex app top bar. They do not constrain the agent: Codex can still choose and run the most appropriate repository command directly from the terminal, including focused build/test/static loops that are not exposed as actions.

Recommended always-visible Codex app actions:

| --- | --- | --- | --- |

## Cleanup Script

Leave the Codex local environment cleanup script empty unless a future workflow introduces a specific reviewed cleanup need. Do not use `docker compose down`, `rm -rf`, or broad deletion commands here.

`external/vcpkg` is a required vcpkg tool checkout, not disposable build output. `vcpkg_installed` is removable only when the operator intends to rerun `tools/bootstrap-deps.ps1`.

## Generated Configuration

The Codex app stores local environment configuration under `.codex` at the project root. The generated file may be checked into Git to share the environment with other operators, but do not hand-edit generated files marked `THIS IS AUTOGENERATED`; update them through the Codex app settings UI.
