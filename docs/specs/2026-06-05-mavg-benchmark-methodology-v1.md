# 2026-06-05 MAVG Benchmark Methodology v1

## Purpose

Define how MAVG evidence will be measured before implementation starts. This document prevents premature claims that MAVG is Nanite-like, production-ready, or superior on any axis. It does not add a benchmark runner, raw result artifact, package recipe, renderer path, or ready claim.

## Status

Active Phase 0 specification for `mavg-research-legal-benchmark-baseline-v1`. This is a methodology baseline only; benchmark execution and superiority claims remain unclaimed until future focused MAVG child plans add measured evidence.

## Benchmark Claim Ladder

Claims must advance in this order:

1. Static clustered asset graph validates deterministically.
2. CPU reference selector produces no-hole selected cluster rows.
3. Conventional indexed-draw fallback renders selected static clusters.
4. GPU culling plus indirect draw produces backend-local evidence.
5. Mesh shader paths produce D3D12 and Vulkan evidence without breaking fallback.
6. Streaming and residency preserve visible fallback ancestors under pressure.
7. Deformation tiers report tight enough bounds and explicit fallback diagnostics.
8. Raster and ray tracing derive payloads from the same cluster graph.
9. Benchmarks show selected axes where MAVG exceeds documented Nanite-like LOD classes.

No child plan may skip a ladder step in its ready claim.

## Source And Asset Policy

Allowed benchmark sources:

- First-party authored meshes.
- First-party generated procedural meshes.
- First-party generated materials and textures.
- Repository sample games and packages.
- Synthetic stress scenes generated by first-party tooling.

Forbidden benchmark sources:

- Unreal Engine sample assets.
- Quixel assets used as Nanite exemplars.
- UE cooked data, extracted data, or reverse-engineered Nanite data.
- Third-party assets without explicit license and repository legal records.

## Required Scene Families

### Scene A - Static Dense Mesh

Purpose:

- Validate cluster graph quality, static selection, fallback quality, and draw/dispatch overhead.

Inputs:

- One first-party dense static mesh family.
- One material partition set.
- One fixed camera path.

Required counters:

- Source triangles and vertices.
- Cluster count, page count, hierarchy depth, resident fallback count.
- Selected cluster count by frame.
- Screen-space error p95/p99.
- Fallback substitution count.
- CPU selection time.
- GPU culling time when implemented.
- Draw, indirect draw, mesh shader dispatch, and primitive counters when implemented.

### Scene B - Environment-Aware Outdoor

Purpose:

- Validate MAVG under existing environment load without broad environment-ready claims.

Inputs:

- First-party static clustered scene.
- Existing selected D3D12 environment evidence path for sky/fog/cloud/rain where available.
- Fixed time-of-day and weather rows.

Required counters:

- All Scene A counters.
- Environment profile id.
- Sky/fog/cloud/rain selected evidence flags.
- Environment-related GPU pass counters already exposed by the engine.
- Explicit non-claims for snow package readiness, volumetric-cloud package readiness, Vulkan/Metal parity, and broad `environment_ready`.

### Scene C - Streaming Pressure

Purpose:

- Validate no-hole behavior under memory and page pressure.

Inputs:

- First-party scene with multiple cluster pages.
- Deterministic camera path that crosses page boundaries.
- Fixed memory budgets.

Required counters:

- Page requests, page hits, page misses, evictions, pinned ancestors.
- Fallback substitutions by reason.
- Page request latency buckets where available.
- Resident CPU bytes, resident GPU bytes, upload/staging bytes, transient GPU bytes.
- Diagnostics for attempts to evict visible fallback ancestors.

### Scene D - Deformation Tier

Purpose:

- Validate static, rigid, skinned, morph, and future dynamic tiers separately.

Inputs:

- First-party static baseline.
- First-party rigid instancing case.
- First-party skinned sample.
- First-party morph sample.

Required counters:

- Deformation tier selected.
- Bounds expansion.
- Refit/update count.
- Conventional fallback count for unsupported tiers.
- Quality diagnostics for conservative bounds.

### Scene E - Raster/RT Consistency

Purpose:

- Validate that raster and ray tracing payloads come from the same cluster graph.

Inputs:

- First-party static cluster graph.
- Backend-local RT feature gate.

Required counters:

- Raster cluster ids.
- RT payload cluster ids.
- BLAS/refit/rebuild policy rows.
- Consistency diagnostics.
- Explicit non-claims when RT feature support is absent.

## Baseline Comparisons

Every benchmark report must include these baselines when the relevant path exists:

- Conventional static mesh renderer.
- Conventional skinned mesh renderer.
- Conventional morph mesh renderer.
- Existing package streaming and residency evidence.
- Existing performance baseline and long-run readiness counters.
- Existing environment evidence rows for selected outdoor tests.

Do not compare against Unreal Engine runtime performance unless a separate legal and methodology review approves the exact data source. The default comparison is against feature classes documented publicly, not against copied UE data.

## Host Evidence Fields

Each raw benchmark run must record:

- Repository commit SHA.
- Branch name.
- Plan id.
- Scene id.
- Package/game target.
- Validation recipe or benchmark command.
- Host OS and version.
- CPU model, core count, SMT state, processor groups, and NUMA node count when available.
- GPU model, driver version, backend, feature level or extension list, timestamp support, and profiler tool.
- Display resolution and render scale.
- Warmup frame count, measured frame count, and deterministic camera path id.
- Thermal/power mode when known.
- All unsupported or host-gated feature flags.

## Metrics

Frame and quality:

- CPU frame p50/p95/p99/max.
- GPU frame p50/p95/p99/max.
- MAVG CPU selection time.
- MAVG GPU culling time.
- Draw/dispatch count.
- Primitive count before and after culling.
- Screen-space error p95/p99.
- Temporal churn rate.
- Fallback rate.

Memory and streaming:

- Resident CPU bytes.
- Resident GPU bytes.
- Upload/staging bytes.
- Transient GPU bytes.
- Page requests, hits, misses, evictions, pinned ancestors.
- Memory diagnostics and budget-pressure status.

Backend:

- D3D12 barrier counts and queue wait rows.
- Vulkan synchronization2 barrier rows and validation status.
- Mesh shader support and limit rows.
- Pipeline statistics support rows.
- Metal host-gated evidence rows when available.

Quality:

- Silhouette error.
- Normal/material error.
- Crack diagnostic count.
- Missing required geometry count. This must be zero for any ready claim.
- RT consistency diagnostic count.

## Pass Conditions

Static CPU selector ready claim:

- Missing required geometry count is zero.
- Every missing fine cluster has a resident fallback ancestor.
- Selection is deterministic for identical inputs.
- Temporal hysteresis prevents repeated oscillation in the fixed camera path.

Conventional renderer ready claim:

- Static clustered geometry renders through the existing draw path.
- Package-visible counters identify MAVG selected clusters and fallback substitutions.
- No public native handles are exposed.

GPU culling ready claim:

- D3D12 and Vulkan have backend-local synchronization evidence for compute-write to indirect/draw-read transitions.
- Null RHI tests provide deterministic counters.
- Fallback path remains available.

Mesh shader ready claim:

- Feature queries and limit rows are recorded.
- D3D12 and Vulkan evidence is backend-local.
- Missing mesh shader support uses compute/indirect fallback with diagnostics.

Streaming ready claim:

- Eviction never removes the last resident fallback ancestor of a visible cluster.
- Missing fine pages degrade quality instead of creating holes.
- Memory pressure rows are deterministic.

Benchmark-exceeds claim:

- The report identifies exact axes, workloads, and hosts where MAVG exceeds a documented feature class.
- The report includes raw measurements, methodology, commit SHA, host data, and non-claims.
- Legal/FTO review is complete before external marketing.

## Artifact Policy

- Raw profiler traces and long benchmark logs are not committed unless intentionally small and reviewed.
- Summaries may be committed under docs only when they are concise, reproducible, and tied to a commit SHA.
- Host-owned artifacts must use stable relative artifact references, not machine-local absolute paths.
- No benchmark data may include proprietary third-party assets or UE data.

## Initial Phase 0 Baseline

Phase 0 records methodology and current evidence only:

- Existing conventional renderer and package evidence remains the baseline until MAVG code exists.
- Existing environment and performance foundation rows are prerequisites for later benchmark scenes.
- MAVG Quality Governor Benchmark Harness v1 (`mavg-quality-governor-benchmark-harness-v1`) now provides renderer-side value rows through `mavg_quality_governor.hpp`, `MavgQualityGovernorResult`, `MavgQualityGovernorCounterRow`, `MavgQualityGovernorLimits`, `evaluate_mavg_quality_governor`, and `has_mavg_quality_governor_diagnostic`, plus the metadata-only `tools/benchmark-mavg.ps1` dry-run harness. The harness emits `mavg-benchmark-harness:` rows with `executes_benchmark=false`, `writes_artifacts=false`, `invokes_profiler=false`, and `native_handles=false`. These rows budget future benchmark evidence and reject unsupported claims, but they are not measured benchmark results.
- Non-claims remain unchanged: no benchmark superiority, Nanite compatibility/equivalence/superiority, broad CPU/GPU/memory optimization, backend parity, Metal readiness, mesh shader execution, deformation readiness, ray-tracing readiness, DirectStorage execution, persistent/autonomous streaming services, or async-overlap proof.
- No current result proves a Nanite-like or Nanite-exceeding claim.
