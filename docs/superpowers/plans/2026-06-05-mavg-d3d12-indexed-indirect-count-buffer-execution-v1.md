**Goal:** Execute D3D12 indexed indirect draws with a CPU-generated upload count buffer through `ID3D12GraphicsCommandList::ExecuteIndirect`, proving `min(count, MaxCommandCount)`, zero-count execution, clamp behavior, and fail-closed count-buffer validation without changing the public RHI contract.

**Architecture:** Reuse the existing public `IndexedIndirectDrawDesc.count_buffer` fields and D3D12 draw-indexed command signature. D3D12 RHI validates both argument and count buffers as CPU-generated upload data using `BufferUsage::indirect | BufferUsage::copy_source`, reads the count value for deterministic stats, decodes only executed commands, but validates the argument range against `MaxCommandCount` to match D3D12 runtime bounds rules.

**Tech Stack:** C++23, D3D12 WARP, Microsoft Learn D3D12 `ExecuteIndirect`, repository PowerShell 7 validation scripts.

## Official Source Audit

Checked on 2026-06-05:

- Microsoft Learn `ID3D12GraphicsCommandList::ExecuteIndirect`: with `pCountBuffer != NULL`, the executed command count is `min(count_buffer_uint32, MaxCommandCount)`; without a count buffer, `MaxCommandCount` is the exact count. Argument and count offsets must be 4-byte aligned, buffers must be buffer resources, and argument/count ranges must remain in bounds.
- Microsoft Learn `ID3D12Device::CreateCommandSignature`: a draw-only command signature may use a null root signature because no pipeline bindings are updated by the signature.
- Microsoft Learn Direct3D 12 Indirect Drawing: command signatures define indirect argument layout, command type, inherited state, and byte stride; command buffers can be generated by CPU or GPU.
- Context7 `/websites/learn_microsoft_en-us_windows_win32_direct3d12`: confirmed D3D12 indirect drawing command-signature examples and sample count-buffer use.

## Scope

In scope:

- D3D12-only `ExecuteIndirect` count-buffer parameter for indexed indirect draws.
- CPU-generated upload count buffer with `BufferUsage::indirect | BufferUsage::copy_source`.
- Count-buffer `uint32` readback for deterministic stats and validation evidence.
- `count == 0` as valid zero execution.
- `count > max_draw_count` clamping to `max_draw_count`.
- `count < max_draw_count` decoding only executed commands while keeping full `MaxCommandCount` argument range validation.
- Count-buffer offset alignment/range/usage/handle fail-closed behavior.

Out of scope:

- GPU-generated count buffers.
- Compute culling dispatch.
- UAV-to-indirect synchronization.
- Public buffer state tracking or public `ResourceState::indirect_argument`.
- Vulkan indirect draw execution, Vulkan `vkCmdDrawIndexedIndirectCount`, Metal ICB, mesh shaders, Work Graphs, Nanite compatibility/equivalence/superiority, and broad optimization claims.
