## Official Source Audit

Checked on 2026-06-05:

- Context7 `/khronosgroup/vulkan-docs`: `vkCmdDrawIndexedIndirect` reads indexed draw parameters from a buffer; multi-draw stride must be a multiple of 4 and at least the command struct size; command buffers must remain in bounds. `vkCmdDrawIndexedIndirectCount` reads a 32-bit count buffer and executes `min(count, maxDrawCount)`. `VK_KHR_draw_indirect_count` is promoted to Vulkan 1.2 but remains feature/support dependent.
- Context7 `/khronosgroup/vulkan-docs`: compute writes feeding indirect draws require synchronization from compute shader write access to draw-indirect/indirect-command-read access.
- Microsoft Learn Direct3D 12 Indirect Drawing (`https://learn.microsoft.com/en-us/windows/win32/direct3d12/indirect-drawing`): command signatures define how argument buffers are interpreted for indirect execution and may be generated by CPU or GPU.
- Microsoft Learn `ID3D12GraphicsCommandList::ExecuteIndirect` (`https://learn.microsoft.com/en-us/windows/win32/api/d3d12/nf-d3d12-id3d12graphicscommandlist-executeindirect`): argument/count offsets are 4-byte aligned and execution is explicit command-list work.
- Microsoft Learn D3D12 resource barriers: indirect argument buffers require the `D3D12_RESOURCE_STATE_INDIRECT_ARGUMENT` state before native indirect execution.

## Scope

In scope:

- Backend-neutral indexed indirect command layout with five 32-bit words: index count per instance, instance count, first index, signed vertex offset, and first instance.
- `IndexedIndirectDrawDesc` over an argument buffer, argument offset, command stride, max draw count, optional count buffer, and count offset.
- Public encode/decode helpers for deterministic tests, package rows, and future GPU culling output.
- Null RHI validation and deterministic stats-preserving execution.
- `BufferUsage::indirect` intent and Vulkan indirect-buffer usage mapping.
- Static docs/manifest/agent checks that keep the ready claim narrow.

Out of scope:

- Native D3D12 `ExecuteIndirect` command signatures or resource-state transitions.
- Native Vulkan `vkCmdDrawIndexedIndirect` / `vkCmdDrawIndexedIndirectCount` command recording.
- Metal indirect command buffers.
- Actual GPU culling compute dispatch.
- Mesh/task/amplification shaders, Work Graphs, deformation, ray tracing, Metal readiness, benchmark superiority, or Nanite compatibility/equivalence/superiority.

## Public API Contract

`engine/rhi/include/mirakana/rhi/indirect_draw.hpp` owns the serial layout:

```cpp
namespace mirakana::rhi {

struct IndexedIndirectDrawCommand {
    std::uint32_t index_count_per_instance{0};
    std::uint32_t instance_count{0};
    std::uint32_t first_index{0};
    std::int32_t vertex_offset{0};
    std::uint32_t first_instance{0};
};

struct IndexedIndirectDrawDesc {
    BufferHandle argument_buffer{};
    std::uint64_t argument_buffer_offset{0};
    std::uint32_t command_stride_bytes{indexed_indirect_draw_command_stride_bytes};
    std::uint32_t max_draw_count{0};
    BufferHandle count_buffer{};
    std::uint64_t count_buffer_offset{0};
};

} // namespace mirakana::rhi
```

`engine/rhi/include/mirakana/rhi/rhi.hpp` exposes:

- `BufferUsage::indirect`
- `IRhiCommandList::draw_indexed_indirect(const IndexedIndirectDrawDesc& desc)`
- `RhiStats::indexed_indirect_draw_calls`
- `RhiStats::indexed_indirect_commands_executed`
- `RhiStats::indexed_indirect_count_buffer_reads`
- `RhiStats::last_indexed_indirect_max_draw_count`
- `RhiStats::last_indexed_indirect_executed_draw_count`
- `RhiStats::last_indexed_indirect_count_buffer_value`

Null RHI execution mirrors direct indexed draw semantics after validation: each decoded command contributes to existing direct draw/indexed draw stats, while indirect-specific stats preserve the draw packet count and count-buffer evidence.
