# Automatic Cooked Package Assembly v0 Implementation Plan (2026-04-29)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a host-independent tool/editor-core contract that turns successful cooked import results into a validated `.geindex` package index, with scene imports reflected as first-class editor asset records.

**Architecture:** Keep cooked package assembly in `mirakana_tools` because it composes import execution output, filesystem reads, dependency edges, and `mirakana_assets` package index serialization. Keep editor changes in `mirakana_editor_core` data models only, without Dear ImGui, SDL3, runtime, renderer, or RHI exposure. Runtime package loading remains unchanged and is used only as proof that generated indexes are shippable.

**Tech Stack:** C++23, `mirakana_assets`, `mirakana_tools`, `mirakana_editor_core`, existing `IFileSystem`, existing unit tests, no new third-party dependencies.

---

## Context

The previous Authored Scene Cook / Package Workflow v0 slice added scene import metadata, scene import actions, scene dependency edges, and canonical `GameEngine.Scene.v1` cooking. The remaining gap is that successful import execution still leaves `.geindex` creation as manual fixture work, and editor imported asset records do not yet promote `AssetImportActionKind::scene` into `AssetKind::scene`.

## Constraints

- Do not add dependencies.
- Do not expose filesystem, SDL3, OS, GPU, RHI, Vulkan, D3D12, Metal, or Dear ImGui handles to game public APIs.
- Keep `mirakana_assets` independent from `mirakana_scene`; package assembly can live in `mirakana_tools`.
- Use existing `AssetImportPlan`, `AssetImportExecutionResult`, `AssetCookedPackageIndex`, and `IFileSystem`.
- Run `pwsh -NoProfile -ExecutionPolicy Bypass -File tools/check-public-api-boundaries.ps1` because public headers change.

## Done When

- `mirakana_tools` can assemble and serialize a cooked package index from a successful import result and import plan.
- Generated indexes reject failed imports, missing cooked artifacts, duplicate imports, unknown dependencies, and edge/entry dependency mismatch.
- A mixed texture/mesh/material/scene package generated by the helper loads through `mirakana::runtime::load_runtime_asset_package`.
- `mirakana_editor_core` registers imported scene artifacts as `AssetKind::scene` and includes scene thumbnail/package rows deterministically.
- Docs, plan registry, roadmap, gap analysis, manifest, Codex/Claude skills, and subagent guidance are synchronized.
- Focused tests, API boundary, schema/agent/format/tidy diagnostics, and `pwsh -NoProfile -ExecutionPolicy Bypass -File tools/validate.ps1` have been run.
