# Sample Desktop Runtime Game

This sample proves the host-gated `3d-playable-desktop-package` foundation for a non-shell `games/<game_name>` executable that can use the reusable Win32 desktop runtime host and be selected by the desktop runtime package lane.

- Implements `mirakana::GameApp`.
- Runs through `mirakana::Win32DesktopGameHost`, which owns Win32 runtime/window setup, event pumping, renderer presentation selection, virtual input/lifecycle state, and `mirakana::DesktopGameRunner` service wiring.
- Uses host-owned `mirakana::Win32DesktopPresentation` and accepts deterministic backend-neutral presentation report fields plus `mirakana::NullRenderer` fallback diagnostics when a native D3D12/Vulkan surface or shader bytecode path is unavailable.
- Supports `--smoke` for a finite Win32 host run registered in CTest.
- Supports `--require-config <path>` for installed package smoke validation of bundled runtime config files through `mirakana::RootedFileSystem`.
- Supports `--require-scene-package <path>` for installed package smoke validation of bundled cooked runtime packages through `mirakana::RootedFileSystem`, `mirakana::runtime::load_runtime_asset_package`, `mirakana::instantiate_runtime_scene_render_data`, and `mirakana::submit_scene_render_packet`.
- Submits a cooked skinned mesh scene with material instance intent, a primary perspective camera, deterministic camera/controller movement, and `mirakana_ui` HUD diagnostics through public `mirakana::GameApp`, scene, UI, and renderer contracts. The host-gated native GPU runtime UI overlay proof routes `mirakana_ui_renderer` box sprites and a cooked texture/atlas-backed image sprite through public `mirakana::IRenderer::draw_sprite` into a renderer-owned final overlay path, with the image binding built from first-party cooked UI atlas package metadata; production text/font/image/atlas/accessibility remains unsupported, and source image decoding plus production atlas packing remain unsupported.
- On Windows desktop runtime package builds, CMake compiles `shaders/runtime_scene.hlsl`, `shaders/runtime_postprocess.hlsl`, `shaders/runtime_shadow.hlsl`, and `shaders/runtime_ui_overlay.hlsl` into target-specific D3D12 DXIL artifacts when `dxc` is available and target-specific Vulkan SPIR-V artifacts when DXC SPIR-V CodeGen plus `spirv-val` are available, and compiles `shaders/environment/height_fog.hlsl` into the selected D3D12 environment-fog DXIL artifact when `dxc` is available.
