# Sample Desktop Runtime Game

This sample proves the host-gated `3d-playable-desktop-package` foundation for a non-shell `games/<game_name>` executable that can use the reusable SDL3 desktop runtime host and be selected by the desktop runtime package lane.

- Implements `mirakana::GameApp`.
- Runs through `mirakana::SdlDesktopGameHost`, which owns SDL runtime/window setup, event pumping, renderer presentation selection, virtual input/lifecycle state, and `mirakana::DesktopGameRunner` service wiring.
- Uses host-owned `mirakana::SdlDesktopPresentation` and accepts deterministic backend-neutral presentation report fields plus `mirakana::NullRenderer` fallback diagnostics when a native D3D12/Vulkan surface or shader bytecode path is unavailable.
- Supports `--smoke` for a finite SDL dummy-driver run registered in CTest.
- Supports `--require-config <path>` for installed package smoke validation of bundled runtime config files through `mirakana::RootedFileSystem`.
- Supports `--require-scene-package <path>` for installed package smoke validation of bundled cooked runtime packages through `mirakana::RootedFileSystem`, `mirakana::runtime::load_runtime_asset_package`, `mirakana::instantiate_runtime_scene_render_data`, and `mirakana::submit_scene_render_packet`.
- Submits a cooked skinned mesh scene with material instance intent, a primary perspective camera, deterministic camera/controller movement, and `mirakana_ui` HUD diagnostics through public `mirakana::GameApp`, scene, UI, and renderer contracts. The host-gated native GPU runtime UI overlay proof routes `mirakana_ui_renderer` box sprites and a cooked texture/atlas-backed image sprite through public `mirakana::IRenderer::draw_sprite` into a renderer-owned final overlay path, with the image binding built from first-party cooked UI atlas package metadata; production text/font/image/atlas/accessibility remains unsupported, and source image decoding plus production atlas packing remain unsupported.
- On Windows desktop runtime package builds, CMake compiles `shaders/runtime_scene.hlsl`, `shaders/runtime_postprocess.hlsl`, `shaders/runtime_shadow.hlsl`, and `shaders/runtime_ui_overlay.hlsl` into target-specific D3D12 DXIL artifacts when `dxc` is available and target-specific Vulkan SPIR-V artifacts when DXC SPIR-V CodeGen plus `spirv-val` are available. Installed package smoke can require `--require-d3d12-scene-shaders` / `--require-d3d12-renderer` or `--require-vulkan-scene-shaders` / `--require-vulkan-renderer` together with `--require-scene-gpu-bindings`, `--require-postprocess`, `--require-postprocess-depth-input`, `--require-directional-shadow`, `--require-directional-shadow-filtering`, `--require-renderer-quality-gates`, `--require-native-ui-overlay`, `--require-native-ui-textured-sprite-atlas`, `--require-gpu-skinning`, and `--require-quaternion-animation`; the host privately builds the scene GPU palette, records the shadow-depth, fixed filtered shadow-receiver scene, sampled scene-depth postprocess pass path, GPU skinning draw path, quaternion scene-transform sampling path, and final native UI overlay, and reports backend-neutral presentation fields plus scene GPU, postprocess, depth-input, directional-shadow, filtering, `renderer_quality_status`, `renderer_quality_framegraph_execution_budget_ok`, `renderer_quality_expected_framegraph_barrier_steps`, GPU skinning, quaternion animation, UI overlay, textured UI overlay, and framegraph counters including `framegraph_passes_executed` / `framegraph_barrier_steps_executed`.
- Proves Lit Material v0, package-visible depth-aware Frame Graph/Postprocess v0, package-visible fixed 3x3 PCF Directional Shadow Filtering v0, Renderer Package Quality Gates v1, D3D12 package GPU Skinning Upload And Rendering v1, and a narrow cooked texture/atlas-backed UI image sprite for the packaged scene path: the cooked skinned mesh carries reviewed tangent-space vertex data plus joint indices/weights, the scene shader samples the base-color texture at mesh UVs, applies material base-color/emissive factors, and evaluates one deterministic directional Lambert light; the directional-shadow smoke path writes a renderer-owned `depth24_stencil8` shadow depth target, samples it in the receiver scene pass through the shadow-receiver descriptor set with fixed 3x3 sampled-depth PCF, then renders scene color and a renderer-owned `depth24_stencil8` scene-depth input into offscreen targets, samples color at `t0/s1` and depth at `t2/s3` in a fullscreen pass, builds the UI image binding from the packaged `hud.uiatlas` metadata record, and reports `postprocess_status=ready`, `postprocess_depth_input_ready=1`, `directional_shadow_status=ready`, `directional_shadow_ready=1`, `directional_shadow_filter_mode=fixed_pcf_3x3`, `directional_shadow_filter_taps=9`, `directional_shadow_filter_radius_texels=1`, `renderer_quality_status=ready`, `renderer_quality_ready=1`, `renderer_quality_diagnostics=0`, `renderer_quality_expected_framegraph_passes=3`, `renderer_quality_expected_framegraph_barrier_steps=15`, `renderer_quality_framegraph_passes_ok=1`, `renderer_quality_framegraph_barrier_steps_ok=1`, `renderer_quality_framegraph_execution_budget_ok=1`, positive `scene_gpu_skinned_mesh_uploads`, positive `scene_gpu_skinned_mesh_resolved`, positive `renderer_gpu_skinning_draws`, positive `renderer_skinned_palette_descriptor_binds`, `ui_atlas_metadata_status=ready`, `ui_atlas_metadata_pages=1`, `ui_atlas_metadata_bindings=1`, `ui_texture_overlay_status=ready`, `ui_texture_overlay_atlas_ready=1`, positive `ui_texture_overlay_sprites_submitted`, positive `ui_texture_overlay_texture_binds`, positive `ui_texture_overlay_draws`, plus `framegraph_passes=3`, `framegraph_passes_executed=6`, and `framegraph_barrier_steps_executed=15` when a native D3D12 or Vulkan presentation is selected. Stable Directional Light-Space Policy v0 is a renderer/scene-renderer planning API, but this package smoke does not consume those matrices yet. This is a narrow smoke proof, not runtime source PNG/JPEG image decoding, hardware comparison samplers, cascades, production atlas packing, editor shadow controls, Metal presentation, production shadow authoring, arbitrary GPU skinning backend parity, or general production renderer quality.
- Does not include SDL3, Dear ImGui, editor, OS, GPU, or backend-native handles in gameplay code. It also does not claim runtime source asset parsing, source image decoding, production atlas packing, material graph, shader graph, skeletal animation production path, arbitrary GPU skinning backend parity beyond the reviewed D3D12 package lane, package streaming, production text shaping, font rasterization, glyph atlases, IME, OS accessibility bridges, Metal readiness, broad generated 3D production readiness, or general production renderer quality.

Build and validate the source-tree desktop runtime lane with:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File tools/validate-desktop-game-runtime.ps1
```

For a finite local smoke run, use:

```powershell
out\build\desktop-runtime\games\Debug\sample_desktop_runtime_game.exe --smoke
```

Package this non-shell proof target with:

```powershell
pwsh -NoProfile -ExecutionPolicy Bypass -File tools/package-desktop-runtime.ps1 -GameTarget sample_desktop_runtime_game
```
