# Sample 2D Desktop Runtime Package

This sample is the host-gated desktop/package proof for the `2d-desktop-runtime-package` recipe.

- Runtime host: `mirakana::SdlDesktopGameHost`
- Gameplay contract: `mirakana::GameApp`, `mirakana_scene`, `mirakana_ui`, `mirakana_audio`, `MK_physics`, `MK_navigation`, `MK_ai`, and public `mirakana::IRenderer`
- Package files: manifest-derived `runtimePackageFiles`, including deterministic data-only tilemap metadata
- Source atlas authoring: `spriteAtlasSourceAuthoringTargets` records reviewed RGBA8 frame rows plus selected page policy, pivot, and slice-border metadata for `plan_sprite_atlas_source_authoring`; `source/assets/package.geassets` and `source/sprites/player_atlas.texture_source` stay outside `runtimePackageFiles`
- Native presentation: owned by runtime host adapters, not game code
- D3D12 package window smoke: packaged `sample_2d_desktop_runtime_package_sprite.vs.dxil` and `sample_2d_desktop_runtime_package_sprite.ps.dxil` loaded through `mirakana::load_desktop_shader_bytecode_pair`, then required with `--require-d3d12-shaders --video-driver windows --require-d3d12-renderer`
- Native 2D sprite package proof: packaged `sample_2d_desktop_runtime_package_native_sprite_overlay.vs.dxil` and `sample_2d_desktop_runtime_package_native_sprite_overlay.ps.dxil` let the host-owned D3D12 RHI path upload the cooked sprite texture and draw scene/HUD sprites with `--require-native-2d-sprites`, including renderer-owned native sprite batch execution counters such as `native_2d_sprite_batches_executed`
- Sprite animation package proof: cooked `runtime/assets/2d/player.sprite_animation` advances through `mirakana::advance_runtime_sprite_flipbook`, applies the sampled frame through `mirakana::sample_and_apply_runtime_scene_render_sprite_animation`, then requires `--require-sprite-animation` with counters such as `sprite_flipbook_frames_sampled` and `sprite_animation_frames_applied`
- Sprite sorting layer package proof: visible scene sprite rows receive distinct `sorting_layer` values in `on_start`, `mirakana::build_scene_render_packet` sorts sprites deterministically, then `--require-sprite-sorting-layer` requires counters such as `sprite_sort_layers_applied` and `sprite_sorted_draws`
- Sprite 9-slice/tiled package proof: visible scene sprite rows in `on_start` configure `draw_mode`, `slice_border`, and tiled `tile_size`; `mirakana::expand_scene_sprite_commands` runs before batch planning and submit; then `--require-sprite-9slice-tiled` requires `sprite_9slice_tiled_source_sprites`, `sprite_9slice_expanded_quads`, and `sprite_tiled_expanded_quads`
- Sprite effects/particles package proof: `mirakana::runtime::plan_runtime_sprite_effect_particles` advances caller-owned transient particle state from selected hit feedback and `mirakana::submit_runtime_sprite_effect_particle_rows` submits top-layer sprite rows after scene sprites, then `--require-sprite-effects-particles` requires counters such as `sprite_effect_particles_spawned_particles`, `sprite_effect_particles_render_rows`, and `sprite_effect_particles_submitted`
- Tilemap runtime UX proof: cooked `runtime/assets/2d/level.tilemap` is sampled through `mirakana::runtime::sample_runtime_tilemap_visible_cells`, then required with `--require-tilemap-runtime-ux` and status counters such as `tilemap_cells_sampled`
- Gameplay systems package proof: deterministic `mirakana::PhysicsWorld2D` contact/trigger rows, `mirakana::plan_navigation_grid_agent_path`, `mirakana::update_navigation_agent`, AI behavior authoring validation, quest/dialogue state transitions plus save-state validation, inventory item pickup/state validation/recipe completion, `mirakana::runtime::plan_runtime_gameplay_interactions` trigger/damage/heal/pickup/objective/feedback/win/loss/restart rows, `mirakana::runtime::plan_runtime_session_profile_resume` save-slot/progression/package-resume evidence, `mirakana::ui::plan_runtime_menu_hud` HUD/menu/dialogue/input-binding prompt rows, reviewed construction placement validation plus runtime-scene placement intent planning, procedural generation plus procedural placement intent evidence, perception blackboard projection, and behavior-tree evaluation are required with `--require-gameplay-systems`, `--require-procedural-generation`, and selected - Audio gameplay mixer package proof: `mirakana::plan_gameplay_audio_mix` plans selected music/SFX bus, pause, fade, loop, trigger, and spatial cue rows over the cooked package audio payload, then `mirakana::AudioMixer` / `mirakana::render_audio_device_stream_interleaved_float` emit deterministic `audio_gameplay_mixer_*` planning, render, and sample diagnostic counters through `--require-audio-gameplay-mixer` without native device ownership or broad codec claims.
- World-region streaming package proof: reviewed package candidates are planned and executed through `mirakana::runtime::plan_runtime_world_region_streaming` plus `mirakana::runtime::execute_runtime_world_region_streaming_safe_point`, then required with `--require-world-region-streaming` and status counters such as `world_region_streaming_load_rows`, `world_region_streaming_unload_rows`, `world_region_streaming_reviewed_package_adoptions`, `world_region_streaming_missing_region_diagnostics`, and `world_region_streaming_safe_point_diagnostics`
- Entity scale/culling package proof: deterministic rows are planned through `mirakana::runtime::plan_runtime_entity_scale_culling`, then required with `--require-entity-scale-culling` and status counters such as `entity_scale_culling_rows`, `entity_scale_culling_visible_rows`, `entity_scale_culling_culled_rows`, `entity_scale_culling_lod_rows`, `entity_scale_culling_projected_draw_cost`, `entity_scale_culling_projected_update_cost`, `entity_scale_culling_budget_protected_rows`, and `entity_scale_culling_budget_diagnostics`
- Scripting sandbox proof: reviewed script module metadata is planned through `mirakana::runtime::plan_runtime_script_sandbox`, then one reviewed entrypoint dispatches through `mirakana::runtime::execute_runtime_script_entrypoint` and a sample-owned deterministic adapter; required with `--require-scripting-sandbox-policy` and status counters such as `scripting_sandbox_entrypoint_rows`, `scripting_sandbox_allowed_permission_rows`, `scripting_sandbox_denied_permission_rows`, `scripting_sandbox_rejected_unsafe_capability_rows`, `scripting_sandbox_budget_diagnostics`, `scripting_sandbox_replay_seed_rows`, `scripting_sandbox_execution_status`, `scripting_sandbox_execution_replay_signature`, and `scripting_sandbox_diagnostics`
- Networking foundation policy proof: reviewed session/trust/transport/replication/replay metadata is planned through `mirakana::runtime::plan_runtime_network_foundation`, then required with `--require-networking-foundation-policy` and status counters such as `networking_foundation_session_rows`, `networking_foundation_transport_rows`, `networking_foundation_channel_rows`, `networking_foundation_rejected_unsafe_transport_rows`, `networking_foundation_replay_prerequisite_rows`, `networking_foundation_security_diagnostics`, and `networking_foundation_diagnostics`
- Simulation orchestration package proof: reviewed fixed-step timing and input-command rows are planned through `mirakana::runtime::plan_runtime_simulation_orchestration`, then required with `--require-simulation-orchestration` and status counters such as `simulation_orchestration_planned_steps`, `simulation_orchestration_command_playback_rows`, `simulation_orchestration_budget_limited_status`, `simulation_orchestration_invalid_command_diagnostics`, and `simulation_orchestration_diagnostics`
- Gameplay authoring review package proof: reviewed gameplay feature rows are checked through `mirakana::review_gameplay_authoring_request`, then required with `--require-gameplay-authoring-review` and status counters such as `gameplay_authoring_review_feature_rows`, `gameplay_authoring_review_accepted_rows`, `gameplay_authoring_review_mutation_ledger_rows`, `gameplay_authoring_review_remediation_rows`, `gameplay_authoring_review_unsupported_claim_diagnostics`, and `gameplay_authoring_review_diagnostics`
- Vulkan package window smoke: toolchain/host-gated packaged `sample_2d_desktop_runtime_package_sprite.vs.spv` and `sample_2d_desktop_runtime_package_sprite.ps.spv`, required with `--require-vulkan-shaders --video-driver windows --require-vulkan-renderer`
