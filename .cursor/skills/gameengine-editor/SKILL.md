## Cursor-only note

Nested prefab refresh rows may expose **`nested_prefab_variant_alignment`** contract labels (`ge.editor.scene_prefab_nested_variant_alignment.v1`) and per-row `nested_variant_alignment.resolution_kind` via `prefab_variant_conflict_resolution_kind_label`, mirroring Prefab Variant conflict `resolution_kind` tooling. Local-child and stale-source subtree refresh rows may additionally expose **`local_child_variant_alignment`** / **`stale_source_variant_alignment`** (`ge.editor.scene_prefab_local_child_variant_alignment.v1`, `ge.editor.scene_prefab_stale_source_variant_alignment.v1`) with per-row `local_child_variant_alignment.resolution_kind` and `stale_source_variant_alignment.resolution_kind`.

Scene prefab instance refresh includes **multi-target batch** review (`plan_scene_prefab_instance_refresh_batch`, `scene_prefab_instance_refresh_batch`, `make_scene_prefab_instance_refresh_batch_action`, and shell strings such as `Review batch prefab refresh` / `Clear batch selection`). **Nested prefab propagation preview** (`propagation_preview`, `ge.editor.scene_nested_prefab_propagation_preview.v1`) can pair with **reviewed chained apply** via `reviewed_chained_prefab_refresh_after_root` / `apply_reviewed_nested_prefab_propagation`, `load_prefab_for_nested_propagation`, and simulation in `validate_nested_prefab_propagation_apply`; batch allows propagation apply for disjoint multi-target sets, while duplicate roots and ancestor/descendant overlaps remain fail-closed. Changing those APIs, retained row ids, or `editor/src/main.cpp` literals must stay aligned with `tools/check-ai-integration.ps1` and `gameCodeGuidance` in the engine agent manifest (edit `engine/agent/manifest.fragments/*.json`, then `tools/compose-agent-manifest.ps1 -Write`)—see the shared editor skills above for full contracts.

**Dear ImGui shell (`editor/src/main.cpp`, `MK_editor`):** follow the **Dear ImGui shell** and **MK_editor Windows and material-preview host cache** sections in `.claude/skills/gameengine-editor/SKILL.md` / `.agents/skills/editor-change/SKILL.md` (varargs vs `string_view`, `TextUnformatted` spans, `std::array` for Combo/scratch floats, single `windows.h` include, `GetEnvironmentVariable` vs `_dupenv_s`, material preview and SDL viewport bridge under `editor/include/mirakana/editor/material_preview_gpu_cache.hpp` / `sdl_viewport_texture.hpp`, public-header IWYU vs `editor/src/material_preview_gpu_cache.cpp` / `sdl_viewport_texture.cpp`, `compile_flags.txt` updates, `check-tidy.ps1 -Files editor/src/main.cpp` plus the focused shell `.cpp` files when touched). `AGENTS.md` Testing lists the same baseline.

**Editor core sources (`editor/core/`):** follow **Editor core C++** in the shared editor skills—`std::string_view::contains` / `starts_with` instead of `find` + `npos`, **Rule of Five** for file-local C API / function-table adapters (often **`= delete` move** with `std::unique_ptr`), **`check-tidy.ps1 -Files` only on `.cpp`** (for header-only edits, pass the primary `.cpp` TU that includes the header or use `clangd` on the `.hpp`), small **`enum class : std::uint8_t`** when **`performance-enum-size`** applies, explicit **`optional.has_value()`** before `->` when **`bugprone-unchecked-optional-access`** fires, **acyclic planner vs nested simulation** for prefab refresh (see **`scene_authoring.cpp`**: `plan_scene_prefab_instance_refresh_without_nested_propagation_simulation`), and `check-tidy.ps1 -Files editor/core/src/<file>.cpp` for narrow TU checks.

**Editor core unit tests:** **`tests/unit/editor_core_tests.cpp`** (`MK_editor_core_tests`) and **Unit tests** in `docs/cpp-style.md`—see **Editor core unit tests** in the shared editor skills; narrow loop `cmake --build --preset dev --target MK_editor_core_tests` / `ctest -R MK_editor_core_tests`.
